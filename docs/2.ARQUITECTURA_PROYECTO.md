# 🏗️ Arquitectura del Proyecto - RRHH Permissions App

Este proyecto está diseñado con una arquitectura **Full-Stack desacoplada**, donde el Frontend (Cliente) y el Backend (Servidor) se comunican mediante una **API REST**.

## 📂 Estructura de Carpetas

### 🟢 Backend (Node.js + Express)
Se encarga de la seguridad, acceso a datos y lógica de negocio.
- `src/config/`: Conexión de base de datos (`database.ts`) y configuración ambiental. Soporte de subidas de Multer en configuración.
- `src/controllers/`: Contiene la **lógica de negocio**.
  - `authController.ts`: Login y perfiles de usuario.
  - `permissionController.ts`: Motor de cálculo de vacaciones que integra saltos inteligentes de festivos e integra la carga de archivos adjuntos (soportes) limitados a 10MB.
  - `rrhhController.ts`: Administración masiva, edición integral de perfiles de colaboradores, gestión de estados (activo/inactivo), estadísticas expansivas para 7 tipos de ausencias y filtrado por **documento de identidad**.
- `src/models/`: Definición del esquema SQL (`schema.ts`).
  - **documento**: Identificador nacional del usuario usado ampliamente y validado en la UI.
  - **activo**: Estado del usuario (1 para activo, 0 para inactivo). **IMPORTANTE**: No borramos usuarios para no perder historial, solo cambiamos su estado a 0.
- `src/routes/`: Definición de los **puntos finales (endpoints)** de la API.
- `src/utils/`: Herramientas auxiliares, como el envío de correos (`email.ts`) y el poderoso calculador de festivos transitorios (`holidays.ts` implementando Ley Emiliani).
- `data/`: Contiene el archivo `.db` de SQLite (Base de datos).
- `uploads/`: Repositorio de almacenamiento unificado para los Soportes Documentales (PDF, PNG, etc).

### 🔵 Frontend (React + Vite)
Es la interfaz visual con la que interactúan los usuarios.
- `src/pages/`: Vistas principales calibradas por Rol.
  - `LoginPage.tsx`: Control de acceso.
  - `DashboardPage.tsx`: Vista general.
  - `PermisosPage.tsx`: Formulario de solicitud colaborador, grid de saldo dinámico y soporte de adjuntos obligatorios o referenciales integrando los 7 tipos de ausencia.
  - `DirectorPage.tsx`: Panel de aprobación para directores con experiencia idéntica a PermisosPage en UI de Solicitud personal.
  - `AdminPage.tsx`: El centro de control corporativo. Incluye soporte de descarga de PDFs directa.
- `src/utils/`: Duplicación del algoritmo de `holidays.ts` en el Frontend para pre-calcular visualmente los días correctos y la fecha de regreso en los Modales antes de enviar peticiones (Mejora UX radical).
- `src/services/` y `src/App.tsx`: Definiciones de enrutamiento y estado global de `UserData`.

---

## 🔄 Flujo de Datos
1. El **Usuario** interactúa con una página en React y adjunta un soporte (vía FormData si corresponde).
2. El **Frontend** envía la petición usando Axios. El cálculo previo ya saltó domingos y festivos nacionales gracias a la utilidad unificada.
3. El **Backend** vuelve a verificar los días (Sanitización) contra la DB, guarda el archivo en `uploads/` si aplica y persiste la transacción SQLite.
4. El **Frontend** actualiza las tablas con botones `<a download>` directos para descargar ficheros.

## 🛠️ Tecnologías Útiles
- **Frontend**: Vite, React, React Router, Axios.
- **Backend**: Express, Better-SQLite3, Multer (Archivos), JWT, Nodemailer, Bcrypt.
